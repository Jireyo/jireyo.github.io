算法专题（SQL）

一、表达式规则

等于

  • 任何值与NULL比较都返回NULL，而不是1或0
  • 不等于：!= 或 <>
  • 安全等于：<=>，两个值都为NULL时返回1，一个值为NULL时返回0

其他

  • between a and b，是一个闭合区间

分组

  • group会将所有null值视为同一项

count

  • count(列名)不统计该列为null的记录
  • count(distinct 列名)可以去重
  • count(1)与count( * )包含null的记录
  • 条件写法：
    • 性质：count(任意数) = 1，count(NULL)=0
    • 条件计数：count(布尔表达式 or null) = count(if(布尔表达式,1,null)) = sum(布尔表达式)
    • 条件比例：avg(布尔表达式) = sum(布尔表达式)/count(1)
    • 条件统计：sum(if(条件,列名,0))

count执行速度区别：

  • 速度上：count(主键)>count(1)≈count(*)>count(其他列)
  • 无主键时：count(1)>count(*)
  • 只有一个字段时：count(*)>count(1)

if

  • if(条件,表达式1,表达式2)，根据条件为1或0取值
  • ifnull(表达式1,表达式2)，如果表达式1=null，返回表达式2，否则返回表达式1
  • nullif(表达式1,表达式2)，如果表达式1=表达式2，返回null，否则返回表达式1
  • isnull(表达式1)，表达式1为null时返回1，否则返回0

union

  • union用于合并多个select的结果，会合并重复项
  • union all不会合并重复项

join

  • cross join就是笛卡尔积，使用on则从该结果集中筛选满足条件的连接行
  • full join可以保证连接同时保留不满足条件的连接行，但mysql中不支持，可以通过left join、union、right join三步实现替换

group_concat

  • 子查询是先查询筛选再连接，需要考虑子查询出的记录数量决定语句编写逻辑，交叉连接则先连接再筛选。

三、API

数字

小数处理

  • round(数值，保留小数位数)，返回浮点数
  • format(数据或字符串，格式)，可以用于格式化日期和小数，处理小数时用法同round，但返回的是字符串

日期

日期差

  • datediff(日期1,日期2)，返回整数天
  • timestampdiff(单位,时间1,时间2)，可以额外指定时间差的单位

日期计算

  • date(日期+1)，加一天，这种用法额外要求日期数据类型为date，不可为字符串，其他函数不受限制
  • date_add(日期,interval 1 day)，加1天
  • date_sub(日期,interval 1 day)，减1天

变量

  • 赋予初值：SELECT @pre := 0
  • 操作：@pre := @pre+1

字符串

正则表达式
菜鸟

  • 用法：WHERE str REGEXP '正则表达式';
  • 注意：空格可以用\s表示，但mysql也要一个\，所以使用\\s表示空格。

字符串拼接
concat(str1,str2,...)

字符串截取

  • 截取左端：left(str,int)
  • 截取右端：right(str,int)

字符串大小写

  • 大写：UPPER(str)
  • 小写：LOWER(str)
