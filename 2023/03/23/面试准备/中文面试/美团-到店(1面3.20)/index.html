  7. MySql

    • 聚簇索引和非聚簇索引的区别（被问了几次了，答得还是不太好）

（1）聚簇索引的各个数据页和页内记录都是按照主键值大小排序的，数据页内的记录按照主键值的大小排成了单向链表，数据页和数据页之间按照主键值的大小顺序排成了双向链表。
（2）聚簇索引的叶子节点存储了完整的用户记录，索引即数据，数据即索引，通过聚簇索引可以直接获取到数据。
（3）非聚簇索引的叶子节点仅存储了索引列+主键，想要获取完整的用户记录，需要在非聚簇索引中获取到符合条件的用户记录后，再根据用户记录的主键信息执行回表操作，即到聚簇索引中查找聚簇索引记录。

  • 覆盖索引是什么？（没听说过这个词。。。）

非聚簇索引中因为不含有完整的数据信息，查找完整的数据记录需要回表，所以一次查询操作实际上要做两次索引查询。而如果所有的索引查询都要经过两次才能查到，那么肯定会引起效率下降，毕竟能少查一次就少查一次。
以上面的age索引为例，它是一个非聚簇索引，如果我想通过年龄查询用户的id，执行了下面一条语句：
select id from userinfo where age = 10;
这种情况是否还有必要去回表？因为我只需要id的值，通过age这个索引就已经能拿到id了，如果还去回表一次不就做了无用的操作了吗？实际上确实是不需要的。索引查询中，如果辅助索引已经能够得到查询的所有信息了，就无需再回表，这个就是覆盖索引。

这里复习下索引类型：
按是否只以主键构建B+树：聚簇索引/非聚簇索引（二级索引）
按二级索引的索引个数来分：普通二级索引/联合索引
按二级索引搜索是否能避免回表：覆盖索引/非覆盖索引
按索引是否唯一（可以为Null）来分：唯一索引（主键索引也是唯一索引）/非唯一索引
