面经：拼多多-平台

一面视频面-平台（3.25，60min）

介绍（2min）

  1. 介绍一下你自己？
  2. 非科班，为什么选择软开？

项目+八股(30min)

  1. 项目有哪些功能？什么模块？分工？
  2. WebSocket在里面做什么用？还有什么其他的作用？
  3. 规则报警怎么做的？
  4. QPS测过吗？有没有想过怎么针对测试结果进行优化？
  5. 为什么不使用多个线程分别处理业务，进行解耦？
  6. 生产过程中出现了什么样的问题，如何解决？

八股（10min）

  1. Java
  2. HTTP与HTTPS的区别？
  3. Mysql索引了解哪些？
  4. 分布式了解吗？推荐MIT的分布式课程和跟着做实验。
  5. 网关有什么作用？

算法(20min)

  • 三数之和（讨论了一下不同要求下的去重）
  • 回溯字符串（秒杀）

反问

  • 主要业务：类似CTO，数据中台。
  • 如何继续学习分布式（建议不要直接框架，多看书）

二面视频面-APP首页推荐（4.1，60min）

介绍(2min)

  1. 介绍一下你自己？
  2. 平时什么时候写代码？怎么学的？

八股1(20min)

  1. 项目中使用了哪些中间件？（答：Redis）
  2. Redis数据结构讲讲？
  3. 跳表结构讲讲？跳表相比二叉树优点在哪里？调表的插入过程？复杂度与二叉树相比哪个好一点呢？
  4. 缓存击穿讲讲？可以怎么解决？
  5. JVM：GC常用算法讲一下。
  6. JVM：GCRoots，脏表工作原理？
  7. JVM：各个堆空间都使用什么样的算法？假设年轻代使用复制算法，如果发现需要清理的垃圾很少，还执行这次GC吗？假设老年代使用的标记压缩算法，压缩过程是如何进行的？（有点深了。。。）
  8. JVM：有什么调优的经验？

算法1(20min)

  1. 有IP段[123.23.23.26,136,46,46,86] 上海这样的几百万条数据，现在给你一个IP，如何查找其所在的地点？（这里ip，string转long，使用treemap找首部、再校验尾部）
  2. 给一个并发场景，多线程统计访问量并打印，找代码中的并发问题：
    1. map -> concurrenthashmap
      1. ConcurrentHashMap，工作原理？
      2. Segement是如何加锁的？如何定位Segment，过程？
    2. map.getOrDefault()和map.put()需要同步执行。
    3. map的打印，打印同时put可能仍在进行，打印结束后需要clear，解决办法有：1.打印时给map加锁，阻止其他线程执行2.双buffer，使用新map做put。旧map打印。

算法2(15min)

  1. 手写冒泡
  2. 冒泡优化
  3. 如何统计交换的次数
  4. 有一位同学使用：求同一个数排序完成前后的位置差，累加，这样做对吗?（不对。有重复计算。）

反问

  • 主要业务：APP首页推荐，并发量最大的部门，技术要求很高。

三面视频面-CTO（4.13）

项目（25min）

  1. 自我介绍
  2. 实习？项目？
  3. 你负责哪一块？
  4. 数据采集怎么做的？
  5. 为什么用MQ？消息掉了怎么办？需要做哪些处理？
  6. MQ持久化，硬盘如果坏了怎么办？

算法（15min）

找所有至少有0，1的区间。
