开发问题

一、Java

jar包和war包的区别

  • jar包：java类集合的压缩包，便于代码统一管理。
  • war包：包含整个web引用程序，是一个可直接运行的web模块，可直接部署在tomcat中发布。
  • ear包：企业引用程序，一般是多个jar包、war包的集合。

springboot项目中对比：

  • 可直接打jar包利用内置tomcat运行，所有tomcat的配置应该在springboot配置文件中修改；
  • 可打包为war包，作为传统的应用交付方式，可灵活选择tomcat版本和修改配置。

同源政策
同源政策

  • 定义：浏览器政策，在一个页面中，浏览器只能访问同源（协议相同、域名相同、端口相同）的资源；
  • 目的：保护cookie，避免cookie共享；
  • 限制范围：主要限制cookie、LocalStorage、IndexDB、dom、ajax请求的发送；
  • 规避方法：不同限制类型都有具体的规避方法，这里主要介绍ajax请求规避同源政策的一种方法：跨域CORS

Nginx代理请求可解决同源政策问题。

跨域
跨域详解

  • CORS需要浏览器和服务器同时支持，具体地
  • 对于简单请求：浏览器发送请求时会增加Origin字段，表示本次请求来源，服务器根据该值判断是否允许访问：
    • 如果不在许可范围，则返回正常HTTP响应，浏览器会对其进行拦截并抛出错误。
    • 如果在许可范围内，响应头上会新增一些字段，如许可范围、是否接受cookie、允许查看的响应头字段。
  • 对于复杂请求（PUT或DELETE请求或Content-Type字段的类型是application/json的请求）：浏览器需要先询问服务器（预检），得到许可后才会发起正式的请求。

ruoyi-server项目结构

  • ruoyi-admin：主程序入口，依赖了framework、quartz、generator模块
    • common:包含==验证码==接口和==文件上传/下载==接口（Service文件在framework中）
    • monitor:包含==缓存监控==接口、==服务器监控==接口、==系统访问记录==接口、==操作日志记录==接口、==在线用户监控==接口。
    • system:包含==参数管理、部门管理、字典管理、登录认证、菜单管理、通告管理、岗位管理、个人信息管理、注册管理、角色管理、用户管理==。（Service文件在System中）
    • tools:有==Swagger接口==访问。
    • resources:包含==i18n、mybatis==等配置。
  • ruoyi-common：主要是一些通用功能。
    • annotation:主要注释
    • config:配置文件，读取yml信息
    • constant:常用常量
    • core:常用基类:==BaseController==、==系统类entity==、==登录对象model==、==AjaxResult==、==TreeEntity==
    • enum:常用枚举
    • exception:包含各种==异常entity==
    • filter:请求过滤器，包括==包装可重复读取inputStream的request==(因为多次读取会报错，我们缓存起来)和==防止xss攻击==
    • utils:常用工具类,包括==对象处理、文件处理、HTML处理、HTTP处理、IP地址获取处理、Excel处理、反射工具类、加密处理、Spring工具类、SQL处理、ID生成器、浮点数、时间、日期、异常、日志、安全、分页、字符串、线程等处理类。==
    • xss:防止xss注入的的==validator==和注解
  • ruoyi-framework：主要为系统层的框架配置，如鉴权、限流、拦截、异常、校验等功能。
    • aspectj:切面，包括==数据权限鉴别，多数据源切换，日志，限流==切面
    • config:配置文件
    • datasource:==多数据源==
    • interceptor:==防止重复提交==的拦截器
    • security: 实现==jwt功能==
    • manager: 异步任务管理器
    • web: 服务器相关信息，==全局异常处理器==，==权限实现服务，登录服务，注册校验方法，token验证处理，用户验证处理服务==
  • ruoyi-generator：代码生成器
  • ruoyi-quartz：定时任务
  • ruoyi-system：主要为system类的vo，mapper，service等，有用户角色等一些服务业务。
  • pom：管理子模块、管理公共依赖，子模块要依赖该公共依赖时，不用再指定版本。

业务功能

分页

  • 使用：startPage()会使下一个sql语句分页。
  • 原理：
    • 从Request中获取分页参数（起始索引，每页记录，排序，排序方向）注入PageDomain实体，调用PageHelper.startPage()；
    • PageHelper.startPage()将分页信息通过setLocalPage存入当前线程中；
    • SpringBoot启动时，为SqlSessionFactory添加了PageHelperInterceptor，其在SQL执行前为其添加分页信息；
  • 注意：
    • 该分页请求参数的获取基于GET请求的路径，所以其他如POST请求若有分页需求，需要将请求参数添加在请求路径上，eg：?pageSize=20&pageNum=1
    • startPage()是一个重载函数，其可接受参数：
      • pageNum(int)：起始页
      • pageSize(int)：每页数量
      • reasonable(bool)：是否允许非法的pageNum和pageSize
      • count(bool)：是否预先执行一次count以得到本查询的结果总数。
      • pageSizeZero(bool)：该值为true时，如果 pageSize=0就会查询出全部的结果，不进行分页。

说明：
  • Mysql中小数实际上只有float 和 decimal两种类型，分别是近似数值和精确数值。double = float(53)，real = float(24) ，numeric = decimal。
  • float存在精度损失，实际上只有float(53)和float(24)两种类型，分别占用8byte和4byte。
  • decimal不存在精度损失，decimal(p,s)中p指定最大位数，s指定小数点右侧的最大位数。

说明：
  • date：YY-MM-DD
  • time：HH:MM:SS
  • datetime：YY-MM-DD HH:MM:SS
  • timestamp：同datetime，可表示的时间范围不同，且受时区影响
