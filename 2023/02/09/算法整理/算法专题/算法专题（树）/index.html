平衡二叉树

定义

  1. 是「二叉排序树」
  2. 任何一个节点的左子树或者右子树都是「平衡二叉树」（左右高度差小于等于 1）

评价指标

  1. 平衡因子BF：左子树和右子树高度差
  2. 最小不平衡子树：距离插入节点最近的，并且 BF 的绝对值大于 1 的节点为根节点的子树。

三、赫夫曼树

定义

  • 二叉树的各节点有权重
  • 带权路径长度 = 叶子节点到根节点的距离×自身权重
  • WPL：所有叶子节点的带权路径长度之和。

构造

每次将根节点权值最小的树合并即可。

四、B树和B+树

B树

定义
B树是一类树，包括B-树、B+树、B* 树等，这是一种专门为外部存储器设计的，如磁盘，它对于读取和写入大块数据有良好的性能，所以一般被用在文件系统及数据库中，其在结构上允许每个节点有更多的子节点。

为什么使用B树做存储？

预备知识

  • 存储时总存在一个存储最小单元（如InnoDB的页）；
  • 主存的存取过程中，通过地址总线传入矩阵后即可从数据总线中得到存储数据，该过程中数据之间的距离对效率没有影响；
  • 磁盘I/O的存取过程中，存在机械运动，每个扇区是磁盘的最小存储单元；
  • 局部性原理：当一个数据被用到时，其附近的数据也通常会马上被使用。

解释

  • 当数据量较大时，内存不够使用，必然需要磁盘I/O；
  • 由于磁盘I/O相对主存过慢，为提高效率，需要减少磁盘I/O，即进行预读；
  • 预读一般以页为基本单位，B树为磁盘位置检索和预读提供了良好的数据结构。

B-树


定义

  • 多路自平衡的搜索树，B树允许每个节点有多个子节点。
  • 关键字分布在整颗树中。

优点

  • 相对二叉树，逻辑上接近的节点物理上也会更近，可以充分利用磁盘预读。
  • 相比二叉树，有效降低了树的高度，且每个节点区确定的范围更精确。

B+树


定义

  • B树的变体，也是多路自平衡的搜索树。
  • 关键字存储在叶子节点上，且叶子节点之间存在链指针。

与B树的区别

  • 结构上，由于关键字存储位置不同，B+树的查询时间复杂度更稳定，B树则取决于搜索关键字的位置；
  • B+树叶子节点两两相连可大大增加区间访问性，可使用在范围查询等，而B-树每个节点 key 和 data 在一起，则无法区间查找。
  • B+树更适合外部存储，由于内节点不保存数据，每个节点占用空间更少，意味着一次可以读出更多的非叶子节点，读出的索引更多，搜索更快。

五、序列化与反序列化

解释
树的反序列化，最需要关注和维护的就是节点之间的父子和子子关系，实现隔离一个父的多个子，其维护的难点在于，遍历子节点时，需要维护其父节点以维护父子关系。

方法
无论使用那种方法，序列化的字符串中每个节点或特殊标记之间需要用空格或逗号等隔开，反序列化时才方便执行：

  • DFS+递归：先序构建字符串，字符串记录节点值后记录子节点值。
    • 序列化时：为了区分出子节点结构，对于二叉树，空节点传"Null"，对于N叉树，固定使用一位记录节点数量即可。
    • 反序列化时：使用一个全局变量代表索引即可；
  • DFS+递归：中序构建字符串，并特殊处理，如二叉树使用(leftNode)root.val(rightNode)序列化，反序列化时，先建立根节点，然后依次、创建左节点、为根节点赋值、创建右节点，其本质与DFS相同；
  • BFS(DFS)+建立映射关系：使用数组记录各节点的值和父节点的索引node.val,parentNode.index;，反序列化时依次实例化并建立关系即可；
  • DFS+迭代+栈：使用root.val(childrens)的方式记录，遇到(将上一个节点入栈，遇到)将节点出栈，栈顶维护了当前父节点；

652. 寻找重复的子树
本题考查二叉树的字符串化：
括号表达法：
node.val + "(" + dfs(node.left) + ")(" + dfs(node.right) + ")"
或
"(" + dfs(node.left) + ")" +node.val+ "(" + dfs(node.right) + ")"

本题开始尝试了以下表示方法，后来发现行不通：
dfs(node.left) + "[" + node.val + "]" + dfs(node.right)

  • 如果node.left没有叶子节点，则dfs(node.left) = [node.left.val]，则dfs(node.left)与[node.val]无法分出哪个是根节点。
  • 所以如果使用括号类标识法，应该保证根节点唯一，左右节点可区分。
