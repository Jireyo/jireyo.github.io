940. 不同的子序列 II
与不同的回文串类似的思路
遍历到索引i的字母c时，根据i-1时的不同子序列：
索引i子序列数量 = A（索引i-1子序列数量，不重复） + B（索引i-1子序列加上c，不重复）-A&B（A中以上一个c结尾的子序列数量）+1（c本身）
dp[i] = dp[i-1] + dp[i-1] - dp[preC-1] +1
分类讨论：

  • 字母c没有出现过，则A&B=0，dp[i] = 2*dp[i-1]+1
  • 字母c出现过，则A中肯定有c本身，末尾的1要剪掉：dp[i] = dp[i-1] + dp[i-1] - dp[preC-1]

813. 最大平均值和的分组
可以分析，我们对数进行k次分组，递推公式为：
dp[k][i] = dp[k-1][j]+avg[j,i];
此时可以判断每次只利用上一次循环比i小的索引，所以只要更新索引时从大到小更新，就可以只使用一个数组判断：
dp[i] = dp[j] + avg[j,i]

临界条件的判断：
当k == 1时，dp[1][i] = pre[i]/i，所以预先处理dp函数后，循环的次数应该减少一次。

1000. 合并石头的最低成本
dp[i][j]表示把[i,j]区间尽可能的合并消耗的最低成本。

  • j-i<k-1时，成本为0
  • j-i=k-1时，必须进行一次合并，出现一个唯一成本
  • j-i>k-1时，必须尽可能地合并，出现一个最低成本并剩余若干元素（不超过k个）

递推时：

  • j-i<k-1时不需要处理
  • j-i=k-1时，进行合并
  • j-i>k-1时，将区间分成两个部分，为了保证尽可能地合并，需要左侧或右侧至少有一侧发生了合并，即长度>=k。所以划分界线时，采取了左侧为一个堆，右侧尽可能合并地的策略。
    • 当左侧只有左端一个元素时，右侧必然发生了尽可能地合并，此时对应左侧元素最后才参与合并的情形。
    • 当左侧有n*(k-1)+1个元素时，表示左侧提前参与了合并的情形。

常用技巧：

  • 元素操作

    • c 插入 A ：A |= (1 << c)
    • A 删除 c ：A &= ~(1 << c)或A ^= (1<<c)

  • 集合间操作

    • A 置空 ：A = 0
    • AB并集 ：A | B
    • AB交集 ：A & B
    • AB子集?：(A & B) == B

    • 补集 ：((1 << n) - 1) ^ A
    • 子集 ：(A & B) == B

    for(int s = set;s;s=(s-1)&set)

    s2 = s^set

2305. 公平分发饼干
一开始打算贪心并使用二分查找的方法，找到刚好可以分给k个孩子的不公平程度。但是孩子们的饼干分配方案是无法贪心的，无法求解。
考虑子问题：
所有零食中，已经有i个孩子分到了零食s集合，则第i个孩子一定分到的是s集合的子集s1，则前（i-1）个孩子的状态为s2（s-s1)。定义dp[i][s]为i个孩子拿到状态s的最小不公平程度：
dp[i][s] = Math.min(dp[i-1][s-s1],sum(s1))
i这个维度可以省略，此时s应该从大到小遍历。

600. 不含连续1的非负整数

dp[i]定义为前i个二进制位，所能产生的合法数字的个数。不出现连续的“1”就是合法数字。
发现有：dp[n] = dp[n-1]+dp[n-2]
即：f(xxxx) = f(0xxx) + f(10xx) => dp(4)=dp(3)+dp(2)

对于小于n的情形，可以拆解n：
f(10100) = f(0xxxx) + f(100xx) = dp(4) + dp(2)
对于连续1情形，处理完该批次即可直接返回：
f(11010) = f(0xxxx) + f(10xxx) = f(11000)，最后面的1在处理第二个1时已经完全考虑。

887. 鸡蛋掉落

  1. 第一种方法：在某一层x扔下一枚鸡蛋，破碎和不破碎无法干预，所以取最糟糕情况（max），而在哪一层丢下，则是可以计算的，所以取外层取min。
  • **dp[i][n]的定义是：使用i个鸡蛋，在n层楼中，确定刚好不破碎点发生在几楼(也可能全排除，此时说明发生在0层)需要的操作数。
  • 显然在x层扔一次消耗一次操作，后续需要在n-x层楼中找刚好不破碎点（或全排除，此时发生在x层）或者在x-1层楼中寻找刚好不破碎点（或全排除，此时发生在0层）。

通过递推公式，可以遍历x求出结果，但应该加速这个过程：
dp[i-1][x-1]和dp[i][n-x]，分别是随x变化而递增/递减的，要使两者的最大值最小，则要求两者的交点（或交点两侧的整数点，这个过程可以通过二分方法加速）

  1. 第二种方法：转化问题为已知鸡蛋数量和操作次数，求最高可以求得的楼层。

初始状态下：已知在0层丢不破碎，未知在1层丢是否破碎。

  • dp[t][k]的含义是，通过t次操作扔k个鸡蛋，可以确定多少楼层丢鸡蛋是否破碎。
  • 显然，在x层扔一次消耗一次操作（并确定在x层丢鸡蛋是否破碎），若不破碎，则可以向上搜索，将x层作为已知的楼层（如0）进行处理；若未破碎，则可以向下搜索，将x层作为已知的必然破碎楼层，搜索下面的楼层是否破碎即可。
