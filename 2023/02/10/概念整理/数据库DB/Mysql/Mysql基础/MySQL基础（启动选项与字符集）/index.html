MySQL基础（启动选项与字符集）

一、启动选项

mysqld

eg:
--skip-networking 禁止网络
--default-storage-engine=MyISAM 选择默认存储引擎

注意：

  • 选项有长形式和短形式
    • 长形式前面--
    • 短形式前面-
    • 一般短形式后面要直接加参数（不能空格）
  • 选项名是区分大小写的

mysqld_safe会间接调用mysqld

配置文件

可以在配置文件中修改启动参数。

版本

我们可以在选项组的名称后加上特定的MySQL版本号，比如对于[mysqld]选项组来说，我们可以定义一个[mysqld-5.7]的选项组，它的含义和[mysqld]一样，只不过只有版本号为5.7的mysqld程序才能使用这个选项组中的选项。

配置文件的优先级

  • 不同配置文件和同一个配置文件中的相同选项，按扫描最后出现的为准。
  • 可以defaults-extra-file指定额外的配置文件，这样会只使用该指定文件。

二、系统变量

系统变量基本概念

  • GLOBAL：全局变量，影响服务器的整体操作。
  • SESSION：会话变量，影响某个客户端连接的操作。（注：SESSION有个别名叫LOCAL）

系统变量基本操作

查看系统变量

SHOW [GLOBAL|SESSION] VARIABLES [LIKE 匹配的模式];

设置系统变量

启动时

  • 启动时指定
  • 配置文件中配置

运行时

设置不同作用范围的系统变量：
SET [GLOBAL|SESSION] 系统变量名 = 值;
或
SET [@@(GLOBAL|SESSION).]var_name = XXX;

注意：

  • 并不是所有系统变量都具有GLOBAL和SESSION的作用范围。
  • 有些系统变量是只读的，并不能设置值。

启动选项和系统变量的区别

  • 大部分的系统变量都可以被当作启动选项传入。
  • 有些系统变量是在程序运行过程中自动生成的，是不可以当作启动选项来设置，比如auto_increment_offset、character_set_client啥的。
  • 有些启动选项也不是系统变量，比如defaults-file。

状态变量

查看状态变量

SHOW [GLOBAL|SESSION] STATUS [LIKE 匹配的模式];

三、字符集

基本概念

字符集

主要字符集：

  • ASCII字符集：128个字符，1字节编码
  • ISO 8859-1字符集：256个字符，1字节编码
  • GB2312字符集：汉字，ASCII中1字节编码，其他2字节编码
  • GBK字符集：对GB2312扩充，1～2字节编码。
  • utf8字符集：兼容ASCII字符集，采用变长编码方式，编码一个字符需要使用1～4个字节。

比较规则

每个字符集都有对应的若干个比较规则，如针对二进制编码比较，大小写比较等等。

MySQL中支持的字符集和排序规则

utf8和utf8mb4

  • utf8mb3：阉割过的utf8字符集，只使用1～3个字节表示字符。
  • utf8mb4：正宗的utf8字符集，使用1～4个字节表示字符。

因为4字节的字符很少，有时可以使用utf8mb3。

字符集的查看

SHOW (CHARACTER SET|CHARSET) [LIKE 匹配的模式];

比较规则的查看

SHOW COLLATION [LIKE 匹配的模式];

后缀 英文释义 描述
_ai accent insensitive 不区分重音
_as accent sensitive 区分重音
_ci case insensitive 不区分大小写
_cs case sensitive 区分大小写
_bin binary 以二进制方式比较

字符集与比较规则的应用

分类

MySQL有4个级别的字符集和比较规则，分别是：

  • 服务器级别：系统变量
  • 数据库级别：创建库时指定
  • 表级别：创建表时指定
  • 列级别：创建表时指定

操作

  • 在创建库/表/列时，若没有指定，默认使用上一级的参数。
  • 只修改字符集，则比较规则将变为修改后的字符集默认的比较规则。
  • 只修改比较规则，则字符集将变为修改后的比较规则对应的字符集。

客户端和服务器通信中的字符集

系统变量 描述
character_set_client 服务器解码请求时使用的字符集
character_set_connection 服务器处理请求时会把请求字符串从character_set_client转为character_set_connection
character_set_results 服务器向客户端返回数据时使用的字符集

MYSQL会使用character_set_client字符集来解码客户端传来的二进制数据，然后将其编码成character_set_connection执行，最后将结果编码成character_set_results返回给客户端。

比较规则的应用

比较规则的作用通常体现比较字符串大小的表达式以及对某个字符串列进行排序中。
